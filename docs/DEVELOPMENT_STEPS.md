# 原生版一步到位开发步骤

## 1. 产品边界

目标不是复制任何成熟磁盘工具的商标、图标或专有实现，而是开发同类型的 Windows 原生磁盘洞察工具：

- 秒级扫描 NTFS 卷。
- 目录树按占用空间排序。
- 大文件列表和文件类型统计。
- 可视化空间占比图。
- CSV/JSON 导出。
- 支持管理员权限下的更完整扫描。

## 2. 技术路线

第一阶段使用 Win32 + C++17 搭建原生桌面程序，并保留清晰模块边界：

- `app`：窗口、控件、消息循环、线程协调。
- `core`：扫描模型、扫描器接口、大小格式化、排序。
- `core::DirectoryScanner`：标准文件系统扫描器，保证功能先闭环。
- `core::NtfsMftScanner`：NTFS 快速扫描器，后续接入 MFT/USN。

第二阶段实现 NTFS 快速扫描：

- 枚举系统卷并识别 NTFS。
- 使用 `CreateFileW("\\\\.\\C:")` 打开卷句柄。
- 使用 `FSCTL_QUERY_USN_JOURNAL` 判断 USN Journal 状态。
- 使用 `FSCTL_ENUM_USN_DATA` 枚举文件引用号、父引用号、文件名与属性。
- 将文件记录按父子关系重建目录树。
- 批量聚合目录大小，避免逐目录递归 IO。
- 对权限失败、离线文件、重解析点和硬链接做独立策略。

第三阶段完善 UI：

- 顶部卷选择、扫描、停止、刷新。
- 左侧目录树，右侧大文件列表。
- 下方类型统计和矩形空间图。
- 扫描进度、耗时、文件数、目录数、错误数。
- 支持列排序、过滤、右键打开目录、复制路径。

第四阶段做工程化：

- 单元测试覆盖大小聚合、排序、过滤。
- 大目录压测和权限异常测试。
- 安装包、签名、版本信息、崩溃日志。

## 3. 推荐里程碑

1. 可启动 Win32 桌面壳。
2. 标准目录扫描可展示根节点。
3. 目录树、大文件列表、类型统计闭环。
4. NTFS 卷枚举和管理员权限检测。
5. MFT/USN 扫描器返回同一份扫描模型。
6. 空间图和导出。
7. 性能优化与安装包。
